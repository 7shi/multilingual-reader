## モデル特性

### MoE vs dense の実効性能

体感では MoE の実効性能は総パラメーター数比で dense の半分以下。
ただし VRAM が十分にある環境では小型 dense モデルと同等の速度で動作するため、
デスクトップ・クラウド前提なら MoE の速度優位は実質的に「小型モデル不要」を意味する。

| モデル | 推定実効品質 |
|--------|------------|
| qwen3.6 (35B MoE) | dense 17b 以下相当 |
| gemma-4-26b (MoE) | dense 10〜13b 相当 |

評価・翻訳など判断力を要するタスクには dense モデルが有利。
MoE は知識量の多さを活かせる用途（用語確認・背景知識補完など）や速度重視の場面に適する。

### CoT と MoE の組み合わせによる影響

| | CoT あり | CoT なし |
|---|---|---|
| **dense** | 判断力高・推論コスト増 | 判断力高・高速 |
| **MoE** | 知識量多・推論コスト増・判断力弱 | 知識量多・高速・判断力弱 |

CoT の有効性はタスク依存であり、翻訳のような暗黙的判断を要するタスクでは
無効化した方が安定するケースがある（Qwen の `--no-think` 実験より）。

### GPT-OSS 120B の特性と活用方針

MoE のため知識量はあるが判断力が弱い印象。加えて **CoT を無効化できない**のが難点であり、
翻訳・評価タスクで CoT の逆効果が生じても回避手段がない。

Groq・Cerebras など高速推論サービスでも利用可能 → 急ぎの用途に柔軟対応できる点は引き続き有効。
