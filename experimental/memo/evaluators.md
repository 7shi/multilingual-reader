# 評価タスク：評価者選定

## 現状の課題：GPT-OSS 120B の天井効果

現在の評価者 GPT-OSS 120B には以下の問題がある：

- **最高点が92点**で頭打ち（100点満点のはずが実質92点上限）
- 90〜92点に全716件中117件（16.4%）が密集
  - 92点: 39件、91点: 42件、90点: 36件
- 92点の39件同士の優劣が判定不能 → 上位モデルの差別化ができない
- 平均78.72点、標準偏差16.73点

この問題により、評価者の正式決定を保留していた。

## 評価者に求める条件

- 天井が高い（95〜100点まで使える）
- 分布が広い（標準偏差が大きく、上位モデルを差別化できる）
- ハルシネーションが少ない
  - GPT-OSS 120B・Gemini 2.5 Flash の既知事例は evaluator_comparison/README.md を参照
  - 新評価者（qwen3.6・gemma-4-31b）でも同種の問題がないか重点チェックが必要

## 中間報告：gemma-4-31b と qwen3.6 の評価特性の比較（2026-04-20）

新評価者候補である `gemma-4-31b` と `qwen3.6` の暫定データを分析した結果、以下の知見が得られた。

### 1. 天井効果の解消とスコア分布
GPT-OSS 120B で見られた「92点の壁」は両モデルとも突破した。
- **gemma-4-31b**: 高得点側に寄りやすく、100点満点を約1.8%（13件）に付与した。減点材料がなければ満点を出す「絶対評価的」な傾向がある。
- **qwen3.6**: 100点は1件のみ。70〜90点台に広く分散しており、最上位モデル間でも微細な差を付ける「相対・審美評価的」な傾向がある。

### 2. 評価精度と「厳格さ」の正体
gemma3-27b による翻訳例（対象ファイル: `tr-cmp/gemma3-27b-2.txt`）で、両者の評価が大きく分かれたケース（Gemma: 100点 vs Qwen: 70点）を検証した。
- **gemma-4-31b** は、表面的な流暢さに引きずられ、専門用語のブレ（fine-tuning を最初は "ajuste fino" と訳し、後で "optimización" に変節）や、文脈の欠落（"In-Context Learning" を "aprendizaje puntual" と誤訳）を見逃して満点を与えていた。
- **qwen3.6** は、CoT を用いて論理的矛盾や用語のブレを厳密に特定し、重大な技術的欠陥として適切に減点（70点）していた。

### 3. 中間結論と CoT の有効性に関する新たな知見
- 今回の翻訳タスク（専門用語を含む Podcast 台本）においては、表面的な流暢さの裏にある「もっともらしい嘘」や技術的不正確さを見抜ける **qwen3.6 の方が評価者としてより信頼できる**ことが示唆された。
- qwen3.6 が減点対象とした「用語の揺れ」は、現在のスライディングコンテキスト方式による忘却（不可抗力）が原因であるが、これは**次期アーキテクチャ（用語集付きサマリー）の導入によって改善すべき課題**として定量的に証明できる指標となる。
- **評価タスクにおける CoT の重要性**: 翻訳という直感・暗黙的タスクにおいては CoT（推論）が逆効果になる現象が確認されていたが、評価という「分析的タスク」においては、Qwen 3.6 が示す通り CoT による論理的な検証が極めて有効に機能している。
- **CoTからの構造化抽出能力**: 評価プロセスにおけるCoT（Thinking）のテキスト自体は散漫になりがちで直接読む対象としては扱いにくいが、qwen3.6は思考過程で発見した具体的な欠陥（混在言語や不自然な直訳の指摘など）を散逸させることなく、最終的なJSONの `reasoning` フィールドに的確な要約として落とし込むことに成功している。この能力により減点根拠の透明性が担保される。

## gemma-4-31b 評価完了後の講評（2026-04-20）

### 2段階評価戦略の検討

gemma-4-31b の甘さを逆手に取り、一次フィルターとして活用する戦略を検討した：

- gemma-4-31b 97点未満（584件 / 82%）→ **問題外として除外**
- gemma-4-31b 97点以上（132件 / 18%）→ 精密評価へ

絶対評価である以上100点が複数出ることは問題ないが、27.2%が95点以上・18%が97点以上という「高得点インフレ」は、評価者としての「甘さ」や「見落とし」を示唆している。一方、gemini-2.5-flash は高得点が絞られており得点分布としては妥当な形をしているが、前述の通り変動が激しく（不安定）、1回の「運の悪さ」で良訳が落とされるリスクが高いため信用しきれない。

したがって、gemma-4-31b は評価が甘く97点以上が多数存在するものの、裏を返せば、その甘い評価者でも97点に届かなかった翻訳は明確な問題ありと判断できる。「97点以上への通過」を最低限の品質保証として使う発想であり、上位モデルの差別化は精密評価に委ねる。

理想としては1つのモデルだけで一定の信頼できるスコアを導き出せるのがベストであるが、モデルの特性（甘さ・ハルシネーション・不安定さなど）によって単独での運用が難しい場合は、このような複合評価（2段階評価戦略など）で補完しあうアプローチも現実的な解として有効である。

精密評価の手段としては、**gemma-4-31b の CoT を有効化する**か、**別の評価者モデルを用いる**かの2択となる。

この132件に対する他評価者の分布を確認したところ、gpt-oss-120b は **99%以上を 80点以上**と評価しており、フィルター精度は高い。

### 外れ値の個別確認結果

gemma-4-31b が 97点以上・他評価者が 70点未満だったケースを個別確認した結果：

**gemma-4-31b が高評価したが実際は評価不能（1件）**：
- `qwen3-30b-tr6-nt-05`：CoT 思考過程が混入した出力は評価不能なフォーマット失敗であり **0点が正しい**。gpt-oss-120b の 0点は妥当。gemma-4-31b は混入テキストから翻訳を拾って 100点を付けており、こちらが誤り。

**gemma-4-31b 側の見落とし（6件）**：
- `aya-expanse-8b-0`：「Tech Éclair」を「Tech Flash」と翻訳（全文での一貫性未確認）→ 見落とし扱い保留
- `aya-expanse-8b-2`：「fine-tuning」を「afilado（削る）」と誤訳、ICL を誤展開 → 見落とし
- `command-r-35b-2-20`：「Tech Éclair」を「Tech Relámpago」と翻訳（全文での一貫性未確認）→ 見落とし扱い保留
- `aya-expanse-32b-2`：原文を大幅に加筆・説明を追加（逐語訳超え） → 見落とし
- `gemma3n-e4b-0-05`：冒頭 Camille の台詞がフランス語のまま未翻訳 → 見落とし
- `gemma3n-e4b-0-10`：同上（履歴数違い） → 見落とし

### gemma-4-31b の評価特性まとめ

今回の評価は CoT を無効化した状態で実施した。gemma-4-31b は壊滅的翻訳の一次フィルターとしては有効だが、**内容の正確性（固有名詞・術語・フォーマット）を要する精密評価には向かない**。詳細は下記「評価者ハルシネーションと『それぞれの正しさ』」を参照。

Gemma 4 は CoT に対応しており、有効化すれば（直感ではなく論理的な検証が入るため）評価精度が大きく向上する可能性がある。今回はコストと速度を優先して CoT 無効化版で検証した。

### 評価者ハルシネーションと「それぞれの正しさ」

外れ値の個別確認を通じて、評価者ごとに固有のハルシネーションが存在することが明らかになった。「ある評価者が低点を付けた＝翻訳が悪い」とは必ずしも言えず、評価者自身のハルシネーションによる誤評価と翻訳の真の欠陥を区別する必要がある。

確定した特性比較は → [evaluator_comparison/README.md](../evaluator_comparison/README.md)「評価者の特性」

2段階戦略（gemma-4-31b で一次フィルター → 精密評価）は、評価者ハルシネーションの影響を構造的に緩和する設計でもある。精密評価には gemma-4-31b の CoT を有効化するか別の評価者モデルを用いることで、gemma-4-31b が見落とす内容エラーを補うことができる。

## 3回評価の変動分析（2026-04-21）

`evaluator_comparison/analyze_eval_variance.py` で評価者ごとの3回評価内変動を定量化した（qwen3.6は評価中のため除外）。

確定した統計表・安定性比較は → [evaluator_comparison/README.md](../evaluator_comparison/README.md)「安定性と精度」

### 統計の解釈

- **レンジ/σ比が全評価者で1.90に一致**。3点正規分布の理論値（≈1.69）より若干高いが、変動の構造は3評価者間で同質であり規模（スケール）のみ異なる。
- **中央値/平均が全評価者で1未満**（右裾が重い分布）。特にgemma-4-31bの乖離が最大（0.74）。多くのアイテムは低変動で少数のアイテムが高変動という右裾型の分布。
- **σのσ/σ平均**：gemma-4-31bが1.00で最も高く二極化が顕著（同点7.3%の一方、失敗翻訳では大きく揺れる）。GPT-OSS 120B・gemini-2.5-flashは約0.78で均一に揺れる傾向。
- **3点サンプルの限界**：1アイテムのσ推定値は自由度2のカイ二乗分布に従い相対標準誤差が約50%。個々のσ値の信頼性は低く、716件の集計で初めて安定する。
