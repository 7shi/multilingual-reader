### 主な問題と観察

**gemma4-26b: 安定して高品質**

3 run とも 94〜97 点。20/ と同等レベルの品質を維持しており、threshold=10 への変更による劣化はない。tr-1・tr-3 は参照訳と同水準（97点）。eval スコアのブレも小さく、安定性が高い。

**qwen3.6-27b: 急落リスクが解消**

20/ では tr-3 で 87 点まで急落したが、threshold=10 では 94〜95 点で安定した。20/ の急落は `prompt` → `indicio` の誤訳が glossary に固定されたことが原因だったが、threshold=10 で要約頻度を上げることで誤訳の固定・強化が抑制されたと考えられる。

**gemma4-e4b: 全 run で低迷（87〜92点）**

20/ では急落 run（tr-2: 68点）があったものの、tr-1・tr-3 は 94〜96 点と高水準だった。threshold=10 では急落は解消されたが、全体的なスコアが低下した。主な減点要因：

- **tr-1（88点）**: fluency=16。`tú` と `vosotros` の二人称代名詞が混在し、自然な流れを損なう。また `sinopsis`（antisèche の訳）は「まとめ」という意味に偏りすぎて「カンペ・メモ」のニュアンスが欠ける。
- **tr-2（87点）**: terminology=16。`grounding` → `conexión a tierra`（電気工学の「接地」）、`bachoter` → `hacer trampas`（「ズル」）の2件の誤訳。20/ の tr-2 と同パターンで glossary 初期蓄積の影響と見られる。
- **tr-3（92点）**: terminology=16（3評価一致）。`grounding` → `puesta a tierra` の誤訳が固定。fluency・readability は改善されているため、術語誤訳が唯一の主要な減点要因。

gemma4-e4b は threshold=10 でも術語誤訳（grounding・bachoter 系）が繰り返されており、glossary の初期蓄積の影響を受けやすい傾向が続いている。

### 総評

- **gemma4-26b** が最安定。94〜97 点の範囲で3回とも高品質を維持し、threshold=20 と同等の結果。
- **qwen3.6-27b** は threshold を短くしたことで 20/ の急落が解消され、94〜95 点で安定。
- **gemma4-e4b** は急落（68点）こそなくなったが、全体的に低迷（87〜92点）。術語誤訳が複数 run にわたって発生しており、このモデルの glossary 運用に課題が残る。
- 20/ との比較では、threshold=10 は急落リスク低減に有効だが、gemma4-e4b の低迷は解消されず、gemma4-26b の最高点（100点）も出なかった。
