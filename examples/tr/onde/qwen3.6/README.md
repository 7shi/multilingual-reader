# examples/tr/onde/qwen3.6/

onde テキストを対象とした追加言語を含む翻訳・評価を行います。

対象言語: [common.mk](../../common.mk) で定義された `CORE_LANGS` + `EXTRA_LANGS`

## 実行

`make` で翻訳・評価・集計を一括実行。翻訳は `tr/`、評価は `evals*/`、スコアは `SCORES*.txt` に出力。

- 翻訳モデル: qwen3.6
- 評価モデル: qwen3.6 (比較用 gpt-oss:120b)
- 設定: threshold=20・keep=5・CoT なし・用語ファイル注入（`../../terms/*-en.{json,tsv}`）

ドイツ語や中国語は意味は十分に伝わりますが、一部の直訳や不自然な表現が残ります。ペルシア語は高度に専門的で流暢ですが、わずかに文体的な修正が推奨されます。セルビア語は意味は十分に通じるものの、「Wow（感嘆詞）」を無関係な名詞に訳したり、突然「Kader（運命）」という単語を挿入するなど、局所的な語彙の幻覚（ハルシネーション）が散見されます。ハンガリー語も同様に意味は通じますが、スペルミスや機械翻訳特有の不自然な表現が含まれています。スウェーデン語もこの帯に入り、内容保持は良好でも自動翻訳らしいぎこちなさが残りました。

**中品質 (60〜79点):** 日本語 (ja), 朝鮮語 (ko), ウクライナ語 (uk), ポーランド語 (pl), ヘブライ語 (he), チェコ語 (cs), トルコ語 (tr)

日本語は意味は通じますが、中国語の単語（直接进入、遵循）や英単語（absolutely fundamental）が不自然に混ざる致命的なハルシネーションが発生しました。朝鮮語は意味は通じますが、英語（podcasts）がそのまま残っていたり、突然漢字（巧妙的）が混入するなどの不安定な挙動が見られます。ウクライナ語・ポーランド語・チェコ語は情報保持は堅実ですが、語法の直訳、語形変化のミス、音声スクリプトとして不自然な言い回しが残ります。ヘブライ語は意味は通じますが、タイポや文法の不一致、生成ノイズ（例：'מmm מm.'）などが残っています。トルコ語は文意は概ね通じますが、直訳による不自然な語彙や、文法的なミスが残っています。

**致命的な欠陥 (60点未満):** インドネシア語 (id), エストニア語 (et), ブルガリア語 (bg), ウルドゥー語 (ur), フィンランド語 (fi), テルグ語 (te), カンナダ語 (kn), アラビア語 (ar), インターリングア (ia), ヒンディー語 (hi), エスペラント (eo), タミル語 (ta)

アラビア語は翻訳中に突然「漢字（姆姆姆）」が大量に混入し、出力が崩壊しています。インドネシア語は極端な直訳調であり、文法・構文が破綻しています。エストニア語は文意は概ね通じますが、直訳による不自然な語彙や、文法的なミスが残っています。ウルドゥー語は致命的な語彙のミス（例：intuitionをغریبیتと訳す）や不自然な構文、謎のキリル文字（ну）の混入などにより破綻しています。ブルガリア語はロシア語とブルガリア語が混ざった不自然な表現（Dobroe doшлите）が出力されたり、意味不明な単語（петел）が混入するなど、語彙のミスが目立ちます。カンナダ語は出力の中にメタな指示文（プロンプトの残骸）が混入するバグが見られました。インターリングアはポルトガル語・スペイン語・英語・ラテン語風の断片が入り混じり、標準 Interlingua として成立していません。エスペラントは文意は概ね通じますが、出力中に突然「漢字」が混入するといった不安定な挙動が確認されました。テルグ語、ヒンディー語、フィンランド語は、スペルミスや無意味な単語の断片が頻発し、ネイティブの読解が困難なレベルに劣化しています。タミル語は qwen3.6 では 18点と、評価ログに「壊滅的な文字化けと綴りミス」が記録されるレベルの出力破損が発生しました。

## 過去の実験: gpt-oss:120b による評価との比較

`qwen3.6` による評価（`evals/`）に加え、比較検証用として `gpt-oss:120b` による評価も実施しました。結果は `gpt-oss-120b/` ディレクトリに保存されています（評価 `gpt-oss-120b/evals/`、スコア `gpt-oss-120b/SCORES.txt`）。

| セルビア語 (sr) | 85 | 79 |

※ 対象言語は当時の8言語のみ。

これらの評価ログを比較すると、両モデルとも**「翻訳テキストに含まれるエラー（プロンプトの混入、漢字の幻覚、致命的な誤訳など）を正確に検知・指摘する能力」については非常に高く、同等に機能している**ことが分かりました。

しかし、そのエラーに対して**「どのように点数をつけるか（採点基準）」において、両者に極端な違い**があり、結果として「自己評価の方が圧倒的に厳しくなる」という興味深い現象が確認されました。

1. **qwen3.6（自己評価）の傾向：容赦ない減点方式（文法警察）**
   `qwen3.6` は自身が生成した翻訳の「文法エラー、正しくない語形変化、スペルミス、不自然な直訳」を正確に検知した上で、「抜本的な書き直しが必要」「完全に破綻している」と自分自身を容赦なく酷評し、**極めて低いスコア（20〜40点台）**をつけています。

2. **gpt-oss:120b（他者評価）の傾向：意味重視の加点方式（寛容さ）**
   対して `gpt-oss:120b` は、漢字の混入や誤訳といった致命的なエラーを正しくログに列挙して指摘しているにもかかわらず、「全体として科学的な内容は伝わる」と文脈のゲシュタルト性を高く評価し、**比較的高めのスコア（60〜80点台）**を与える傾向がありました。
